# Lesson

The final project is not about adding many new features. It is about arranging the pieces into a maintainable agent architecture.

## What changes in this step

- Modules are organized by responsibility.
- Configuration becomes explicit.
- Demo and smoke tests prove the full path works.

## Key idea

A Hermes-like agent is a system of boundaries: provider, context, tools, state, memory, skills, scheduler, gateway, and delegation.
